## 二、页面详细设计

### 1️⃣ 赛事信息页

**小组赛（看板模式）**
- 12 个小组（A-L）以卡片网格排列，每小组一张「看板」
- 每张看板包含：小组名、四队积分榜（排名 / 队名 / 赛 / 胜 / 平 / 负 / 进 / 失 / 净 / 分）
- 每个小组的已赛结果列表
- 小组出线线直观标注
- 颜色标识：晋级区/淘汰区

**淘汰赛（树形图 / 晋级图）**
- 32 强 → 16 强 → 8 强 → 半决赛 → 决赛 + 季军赛
- 经典对阵树形图，左侧球队，右侧晋级
- 已完赛标注比分，未赛显示时间/场地
- 重点关注已晋级的球队路径

### 2️⃣ 数据看板

- 🏆 **积分总览**：12 组积分横向对比
- ⚽ **射手榜**：金靴奖竞争实时排名
- 📊 **球队统计**：进球/失球/控球率/射门等
- 🔥 **冷门指数**：爆冷比赛标记（如西班牙 0-0 佛得角）
- 📈 **小组出线概率**（可选）

### 3️⃣ 资讯博客

- 文章列表页：按时间倒序排列
- 文章详情页：图文混排
- 标签分类：战报 / 深度 / 数据 / 花絮
- 考虑到赛事进行中，内容手动更新

## 三、技术方案

| 项目 | 选择 |
|------|------|
| 前端 | 纯 HTML + CSS + JavaScript（零依赖，快速部署） |
| 样式 | 暗色主题，世界杯品牌配色（深蓝/金/白） |
| 数据 | 静态 JS 模块，手动更新（赛事期间每天更新比分） |
| 部署 | 可直接通过 GitHub Pages / Vercel / 任意静态托管 |
| 响应式 | 移动端优先，适配手机/平板/桌面 |
